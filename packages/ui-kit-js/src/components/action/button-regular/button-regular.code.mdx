# Button Regular

## Introduction
All button types can be generated out of the basic button by implementing additional modifier classes or React proprties.

### Buttons as button elements
Buttons used as `<button>` elements to trigger actions.

### Regular basic (default)
If using basic type, no extra class/prop declaration is needed.

<Example noTheme={false} className="sg-example-button-regular">
  {(theme) => {
    const setTheme = theme === "inverted" ? true : false
    return (
      <React.Fragment>
        <p-button-regular inverted={setTheme}>Click Here!</p-button-regular>
        <p-button-regular inverted={setTheme} disabled="true">Click Here!</p-button-regular>
      </React.Fragment>
    )
}}
</Example>

### Regular loader state
If a button has a loading state, it must also be set in a disabled state to prevent click behaviour caused by the user.
To show the loader icon, the default svg icon must be replaced with the loader component

<Example noTheme={false} className="sg-example-button-regular">
  {(theme) => {
    const setTheme = theme === "inverted" ? true : false
    return (
      <React.Fragment>
        <p-button-regular loading="true" inverted={setTheme}>Loading...</p-button-regular>
        <p-button-regular variant="ghost" loading="true" inverted={setTheme}>Loading...</p-button-regular>
        <p-button-regular variant="highlight" loading="true" inverted={setTheme}>Loading...</p-button-regular>
      </React.Fragment>
    )
}}
</Example>

### Button regular with specific icon
If another icon needs to be implemented, just replace the default icon with your new icon.

<Example noTheme={false} className="sg-example-button-regular">
  {(theme) => {
    const setTheme = theme === "inverted" ? true : false
    return (
      <React.Fragment>
        <p-button-regular icon="icon_phone.min.svg" inverted={setTheme}>Click Here!</p-button-regular>
      </React.Fragment>
    )
}}
</Example>
