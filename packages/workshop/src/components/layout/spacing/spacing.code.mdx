# Spacing

## Introduction
A component to add margins and paddings to elements.

### React specific hints
If this component has only one child, those classes are added directly to the child using the className prop to avoid unnecessary wrapper divs. This only works if the child component supports className (which it totally should anyway) and if the number of children isn't dynamic and doesn't change. If this is not the case, set the "wrap" property to manually create a wrapper div.

#### Usage
Static spacings can be used as `padding` or `margin` as a property of the `<Spacing>` component and to define any (S)CSS property with a variable.

**Given values are:**

`"auto" | 0 | 4 | 8 | 16 | 24 | 32 | 40 | 48 | 56 | 64 | 72 | 80`

**Inside SCSS you can use the following variables:**

* `$p-spacing-0`
* `$p-spacing-4`
* `$p-spacing-8`
* `$p-spacing-16`
* `$p-spacing-24`
* `$p-spacing-32`
* `$p-spacing-40`
* `$p-spacing-48`
* `$p-spacing-56`
* `$p-spacing-64`
* `$p-spacing-72`
* `$p-spacing-80`

**Example**:
```
.lorem-ipsum {
  margin-top: $p-spacing-24;
}
```

#### Usage
Responsive spacings can be used to achieve different spacings on different viewport sizes, e.g. to equalize overall whitespace between elements.

**Given values are:**

`"a" | "b" | "c" | "d" | "e" | "f" | "g"`

**Inside SCSS you can use the following mixins:**

* `@include p-spacing-a("type", "direction", "sign")`
* `@include p-spacing-b("type", "direction", "sign")`
* `@include p-spacing-c("type", "direction", "sign")`
* `@include p-spacing-d("type", "direction", "sign")`
* `@include p-spacing-e("type", "direction", "sign")`
* `@include p-spacing-f("type", "direction", "sign")`

**Example**:
```
.lorem-ipsum {
  @include p-spacing-a("margin", "top" ,"+")
}
```
