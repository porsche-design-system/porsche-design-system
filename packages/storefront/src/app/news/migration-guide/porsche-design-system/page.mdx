## Migrate from Porsche Design System v3 to v4

With the new **Porsche Design Language** comes some changes regarding layout and design principles. To keep refactoring
efforts as low as possible when upgrading from `v3` to `v4`, **breaking changes** were avoided as far as possible.
Nevertheless, there are a few breaking changes and some more deprecations which should receive attention.

## Notable New Features

- New design language for all components
  - New color palette
  - Redesigned form elements and buttons
  - Redesigned footer of dialog elements
  - All headings use regular as font-weight
- Design Tokens are exposed as CSS variables (currently colors only).
  - [Colors](/styles/color/examples/)
- Theming now works by adding a `light`, `dark` or `auto` CSS class to any HTML element
  - [Vanilla JS](/developing/vanilla-js/advanced/#global-theme)
  - [Angular](/developing/angular/advanced/#global-theme)
  - [React](/developing/react/advanced/#global-theme)
  - [Vue](/developing/vue/advanced/#global-theme)
  - [Next.js](/developing/next-js/advanced/#global-theme)
  - [React Router](/developing/react-router/advanced/#global-theme)

## Planned upcoming changes in next alpha release

- Redesign `p-accordion`, `p-tabs`, `p-tabs-bar` and `p-pagination`
- Overhaul storefront documentation and examples
- Introduce design language guidelines (Brand Guide)
- Expose more design tokens as CSS variables (e.g. spacing, typography, shadows, etc.)
- Review `light-dark()` and `color-scheme` and refine the theming strategy as appropriate
- Evaluate support for streaming React Server Components using suspense

## üõ†Ô∏è Setup Changes

### Partials

- Removed `getInitialStyles()`, `getFontFaceStyles()` and `getFontFaceStylesheet()` partial in favor of CSS imports from
  `@porsche-design-system/components-{js|angular|react|vue}`.

```diff
- import { getInitialStyles, getFontFaceStyles, getFontFaceStylesheet } from '@porsche-design-system/components-{js|angular|react|vue}/partials';
```

- Instead, a mandatory CSS global style, including CSS color variables, font-face definitions and normalize styles needs
  to be imported into your main css file. See [Styles](/styles/introduction/) for more information.

```diff
/* src/style.css */
+ @import '@porsche-design-system/components-{js|angular|react|vue}/index.css';
/* Alternative with fonts from China CDN */
+ @import '@porsche-design-system/components-{js|angular|react|vue}/cn/index.css';
```

- If you need to keep the border-radius aligned with the Porsche App, you can enable it by importing a separate CSS
  file.

```diff
/* src/style.css */
+ @import '@porsche-design-system/components-{js|angular|react|vue}/legacy-radius.css';
```

- Automatic FOUC handling, previously defined in `getInitialStyles()`, must be implemented manually now. See Getting
  Started for more information:
  - [Vanilla JS](/developing/vanilla-js/getting-started/#integration)
  - [Angular](/developing/angular/getting-started/#integration)
  - [React](/developing/react/getting-started/#integration)
  - [Vue](/developing/vue/getting-started/#integration)
  - [Next.js](/developing/next-js/getting-started/#integration)
  - [React Router](/developing/react-router/getting-started/#integration)

```diff
// Vanilla Js, React, Vue
+ :not(:defined) {
+   visibility: hidden;
+ }

// Angular (Uses custom elements which are never defined like app-root, so we need to manually list the components)
+ :is(p-accordion, p-banner, p-button, prefixed-p-button,...):not(:defined) {
+   visibility: hidden;
+ }

// Next.js, React Router
+ :not(:defined,[data-ssr]) {
+   visibility: hidden;
+ }
```

- Removed `getBrowserSupportFallbackScript()` and `getCookiesFallbackScript()` which are now in separate repo
  `@porsche-design-system/fallbacks` coming soon.

```diff
- import { getBrowserSupportFallbackScript, getCookiesFallbackScript } from '@porsche-design-system/components-{js|angular|react|vue}/partials';
```

- Removed `getDSRPonyfill()` partial as it is no longer needed.

```diff
- import { getDSRPonyfill } from '@porsche-design-system/components-{js|angular|react|vue}/partials';
```
