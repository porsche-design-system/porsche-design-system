## Usage

The following segment provides instructions for designers and developers regarding the appropriate utilization of this
component in various situations.

`p-input-number` is intended for true numeric input scenarios, especially when increment and decrement controls enhance
the user experience. It should not be used for values that merely happen to consist of digits â€” such as postal codes or
credit card numbers. For those cases, consider using another input with an appropriate input mode. For further guidance,
see the
[MDN documentation on number inputs](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/number#using_number_inputs).

### Do:

- Use when a numeric value is required or optional.
- Use without a label only when the context is clear.
- Use for validation and error of input.
- Use asterisk for required fields.

### Don't:

- Don't use labels with more than two words whenever possible (keep it short and descriptive).
- Don't use instructional or description text for the label.
